Introdução a APIs e API Gateway

API

API (Application Programming Interface) conjunto de regras e definições que permite que softwares comuniquem entre si,
possibilita a integração de diferentes sistemas ou plataformas, promovendo a troca de dados e funcionalidades.

Exemplos de API no dia a dia

Serviços de mapas: Aplicativos de entrega ou transporte usam APIs de serviços de mapas (como Google Maps) para buscar informações de geolocalização e rotas.

Redes Sociais: Aplicativos terceiros podem utilizar APIs do Facebook ou Instagram para postar, buscar ou interagir com dados.

Tipos de API:

  • APIs REST: Seguem princípios do protocolo HTTP, permitindo que clientes façam requisições para recursos específicos via URL.

  • APIs SOAP: Usam XML e são mais estruturadas, porém mais complexas.

  • GraphQL: Permite consultas mais flexíveis, onde o cliente define quais dados quer receber

    • Benefícios de uma API

    Integração fácil : Permitem que diferentes sistemas troquem dados de maneira simples e eficiente.

    Automação: Processos que antes eram manuais podem ser automatizados usando APIs.

    Escalabilidade: Empresas podem ampliar ou diversificar seus serviços, oferecendo APIs para parceiros e clientes.

    Segurança: APIs podem controlar quais dados e funções são acessíveis, garantindo proteção e privacidade.

    API Gateway

    API Gateway é um servidor intermediário que fica entre os clientes (como aplicativos ou usuários) e uma coleção de serviços backend.

    Função Principal

    Gerenciamento centralizado de APIs:
    O Gateway atua como ponto de entrada único para todas as APIs de um sistema. Isso facilita o controle, segurança e monitoramento das APIs.

    Principais Responsabilidades:

    Autenticação e Autorização: Valida quem pode acessar as APIs e garante que apenas usuários autorizados tenham permissão.

    Rate Limiting (Limitação de Requisições): Limita o número de requisições que um cliente pode fazer em um período, protegendo o sistema contra sobrecarga.

    Roteamento de Solicitações: Direciona as requisições dos clientes para o serviço adequado no backend.

    Agregação de Serviços: Pode combinar dados de múltiplos serviços backend em uma única resposta para o cliente.

    Exemplo

    Um aplicativo de e-commerce que tem diferentes APIs para gerenciar estoque, pagamento e entrega.

    Documentação do projeto: Dreambuilder

    A Dreambuilder é uma aplicação web que consiste em ajudar users a realizar/concretizar metas/sonhos.
    Consoante a meta e o tempo disponível, a Dreambuilder vai construir um plano que permita a realização dessa meta.

    Esta aplicação permite também a comunicação entre users, seja individualmente através das amizades na app, ou através de grupos, o que ajuda a motivarem-se uns aos outros, partilha de experiências, troca de dicas/ideias e crescimento de grupo.

    A Dreambuilder está no dia a dia do user, o que faz o plano ser atingido na meta escolhida (data),ou seja o user tem que registar os seus dados diariamente, criando um "streak". Para além disso envia notificações com frases motivacionais e lembretes. Para concluir a Dreambuilder faz uma análise semanal, com os dados registados ao longo da semana e através de uma AI fala com o user para rever o plano e se for preciso fazer alterações.
